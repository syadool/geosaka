# 要件定義書: GeOsaka — GeoGuessr 大阪版 Web サイト

作成日: 2026-07-11 / 作成者: Fable（要件定義担当）

## 1. 概要

大阪府内の実在スポットの写真を見て「ここはどこか」を地図上で推測し、実際の位置との距離に応じてスコアを競う、GeoGuessr ライクなシングルプレイヤー Web ゲーム。ブラウザだけで完結し、APIキー・課金・アカウント登録を一切不要とする。

## 2. 背景と制約（重要な前提）

- **Google Street View API は不採用（MVP）**: APIキーと課金登録が必須のため。代わりに **Wikimedia Commons 等の自由ライセンス写真をキュレーションした「写真パネル方式」** を採用する。
- 画像は Wikimedia Commons へのホットリンク（upload.wikimedia.org は外部埋め込みを公式に許容）で表示し、**ライセンス・作者のクレジット表記をラウンド結果画面に表示**する（推測前に出すと地名が漏れてネタバレになるため、結果表示後にクレジットを出す）。
- 将来 Street View に差し替えられるよう、**画像供給部分（SceneProvider）は抽象化**しておくこと。
- バックエンド不要の静的サイトとして構築する（ホスティングは任意の静的ホスティングを想定）。

## 3. 機能要件

### 3.1 ゲームフロー
1. **タイトル画面**: ゲーム開始ボタン、遊び方の簡単な説明。
2. **ラウンド（1ゲーム = 5ラウンド）**:
   - 大阪府内スポットの写真を大きく表示（ズーム/パン可能だと望ましい。最低限、クリックで拡大）。
   - 画面内に推測用の地図（大阪府周辺を初期表示）。地図をクリックしてピンを置き、「推測を確定」ボタンで回答。
   - 制限時間は MVP では設けない（設計上、後から追加可能に）。
3. **ラウンド結果画面**:
   - 推測地点と正解地点を地図上に線で結んで表示。
   - 距離（km/m）とラウンドスコアを表示。
   - スポット名・簡単な解説・写真クレジット（作者/ライセンス/出典リンク）を表示。
4. **最終結果画面**: 5ラウンド合計スコア、各ラウンドの内訳、「もう一度遊ぶ」ボタン。

### 3.2 スコアリング
- GeoGuessr 準拠の距離減衰方式: `score = round(5000 * exp(-distance_km / k))`。
- 対象が大阪府（東西南北 約60km 圏）なので減衰係数 k は狭域向けに調整する（例: k = 3〜5。詳細設計で確定）。
- 満点条件: 誤差 100m 以内は 5000 点。
- 距離計算はハバースイン（Haversine）公式。

### 3.3 スポットデータ
- 大阪府内の **15箇所以上** をキュレーションし、静的な JSON/TS データとして同梱する。
- 各スポットのデータ項目: id、名称（日本語）、緯度経度、写真URL（Wikimedia Commons）、写真クレジット（作者・ライセンス名・出典URL）、一言解説、難易度タグ（任意）。
- 有名どころ（道頓堀、大阪城、通天閣、梅田スカイビル、住吉大社、万博記念公園 等）と、やや難しい場所（郊外の駅前・商店街等、自由ライセンス写真が確保できる範囲で）を混在させる。
- 1ゲームの5箇所はプールからランダム非復元抽出。

### 3.4 地図
- 推測用地図・結果表示地図ともに **APIキー不要の地図ライブラリ + OpenStreetMap タイル** を使用（技術選定フェーズで確定。Leaflet を有力候補とする）。
- OSM のタイル利用ポリシーに従い attribution を表示する。

### 3.5 UI/デザイン要件
- **frontend-design 品質基準**: 生成AIっぽい没個性なデザインを避け、大阪らしさ（ネオン・道頓堀・コテコテ感、ただし品位を保つ）を反映した独自性のあるビジュアルにする。配色・タイポグラフィに意図を持たせること。
- 日本語 UI。フォントは Web フォント（Google Fonts 等のキー不要 CDN）可。
- レスポンシブ対応: デスクトップ優先だがモバイル（375px幅）でも操作可能なこと。
- 写真読み込み中のスケルトン/ローディング表示。画像読み込み失敗時は代替スポットに差し替えるかエラーメッセージを出す。

## 4. 非機能要件

- **APIキー・課金・サーバー不要**。ビルド成果物は静的ファイルのみ。
- 外部依存はライセンス上問題のないもの（OSM タイル、Wikimedia 画像、npm の OSS ライブラリ）のみ。
- 主要モダンブラウザ（Chrome/Edge/Firefox/Safari 最新）で動作。
- Lighthouse を意識した基本的なパフォーマンス配慮（画像の遅延読み込み等）。
- コードは TypeScript を推奨（技術選定フェーズで確定）。lint / build が通ること。

## 5. スコープ外（MVP に含めない）

- マルチプレイ、ランキング、アカウント、サーバーサイド機能
- Street View 実写パノラマ（差し替え可能な設計のみ担保）
- 制限時間、移動（GeoGuessr の「歩き回り」）機能
- 多言語対応（日本語のみ）

## 6. 受け入れ基準

1. `npm run build`（相当）が警告なく成功し、静的ファイルが生成される。
2. タイトル→5ラウンド→最終結果→再プレイ の一連のフローが操作できる。
3. 地図クリックでピンが置け、確定後に正解との距離とスコアが正しく計算・表示される（誤差100m以内=5000点、距離増で単調減少）。
4. ラウンド結果に写真クレジットが表示される。
5. APIキーや環境変数なしで、クローン直後に `install → dev` で動く。
